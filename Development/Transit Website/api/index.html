Intro to Transit Pakistan API

You can communicate with Transit Pakistan using it's XML based API. Transit Pakistan will provide some response containing suggestions about locations, routes and vehicles. Currently our API is under development so we will only provide coverage to selected routes in Karachi city. Below we will describe how you can communicate with Transit Paistan API.


Transit Pakistan API is divided into three parts.

  1. Locations Data
  2. Routes Data
  3. Vehicles Data
  4. SMS Service
Following are the details of these parts and how to communicate with them.

1. Locations Data

This part of our API will allow you to access the names of locations that are currently in our coverage. The following snippet shows you how to invoke locations api parts. 

Post: http://api.transit.pk/CITY/locations/
Headers: Content-Type: application/XML
Response: [Sample Response from http://api.transit.pk/karachi/locations/]
										
<Locations>
<LocationName>7th Day Hospital, Karachi</LocationName>
<LocationName>Abdullah College, Karachi</LocationName>
<LocationName>Agha Khan Jamat Khana, Karachi</LocationName>
.
.
.
</Locations>
* - CITY has to be replaced by the city for which you want locations which are covered by Transit Pakistan in that city.

As of May 2016, Transit Pakistan only provides coverage in Karachi, Pakistan. So the CITY attribute in your post will be 'karachi' in small alphabets because it is case sensitive. In the above response each new location is wrapped inside <LocationName> attribute. All <LocationName> attributes are then merged under <Locations> attribute which act as a document root.

Each <Vehicle> attribute has its own set of attribute that can be interpreted to get more information about that particular vehicle on this route. All <Vehicle> attributes are then merged under <Route> attribute which act as a document root.

4. SMS Service

This part of our API will allow you to access the same data via SMS service that you can access via our online API. The following snippet shows you how to invoke SMS api parts. This part of API only supports route data. 

Format: City: CITY From: STARTINGPOINTNAME To: ENDINGPOINTNAME
Response: [Sample Response of City: Karachi From: Sadar To: Tower]
										
Following vehicles will take you from Sadar to Tower
4-K - 15 Rs
4-L - 15 Rs
5-C - 15 Rs
6 - 15 Rs
6-B - 15 Rs
UTS-12 - 15 Rs
B-5 - 15 Rs

									
* - CITY has to be replaced by the city for which you want routes which are covered by Transit Pakistan in that city.
* - STARTINGPOINTNAME has to be replaced by name of Starting Point for which data is required.
* - ENDINGPOINTNAME has to be replaced by the Destination name to where you want to go.

As of May 2016, Transit Pakistan only provides coverage to selective 15000+ routes combinations in Karachi, Pakistan.
